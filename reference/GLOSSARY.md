# Three.js 3D 开发 — 术语表

> 本术语表是所有课件中概念的统一参考。每个课件中的术语解释必须与此表一致。

---

## 核心概念

| 术语 | 英文 | 定义 | 类比 |
|------|------|------|------|
| 场景 | Scene | 容纳所有 3D 物体的容器，所有物体必须添加到场景才能被渲染 | 🎭 舞台 |
| 相机 | Camera | 定义观察者的位置和视角，决定画面中能看到什么 | 👁️ 观众的眼睛 |
| 渲染器 | Renderer | 将 3D 场景"画"到屏幕上的引擎，把 3D 数据变成 2D 像素 | 🖌️ 画笔 |
| 网格 | Mesh | Three.js 中所有可见 3D 物体的基本类型 = Geometry + Material | 🏺 花瓶 |
| 几何体 | Geometry | 定义物体的形状结构（顶点位置、面、UV 坐标等） | 🏺 花瓶的形状 |
| 材质 | Material | 定义物体的外观（颜色、光泽、透明度、纹理等） | 🎨 花瓶的釉色 |
| 纹理 | Texture | 贴在 3D 模型表面的 2D 图片，用于增加细节 | 🖼️ 贴纸 |
| 光源 | Light | 照亮场景的发光源，无光则场景全黑 | 💡 灯泡/太阳 |
| 阴影 | Shadow | 物体遮挡光线后在另一物体上产生的暗区 | 🌑 影子 |

---

## 几何体

| 术语 | 英文 | 定义 |
|------|------|------|
| 顶点 | Vertex | 3D 空间中的一个点，由 x, y, z 坐标定义 |
| 面 | Face | 由 3 个顶点连接而成的三角形，所有 3D 模型最终都由三角形组成 |
| 边 | Edge | 两个顶点之间的连线 |
| 法线 | Normal | 垂直于面的方向向量，用于光照计算 |
| 分段数 | Segments | 几何体表面的细分程度，越高越光滑但性能越差 |
| 线框 | Wireframe | 只显示几何体的边线，不显示面，用于调试 |
| 包围盒 | Bounding Box | 包围几何体的最小长方体，用于碰撞检测和视锥剔除 |

---

## 渲染管线

| 术语 | 英文 | 定义 |
|------|------|------|
| 渲染管线 | Rendering Pipeline | GPU 将 3D 数据转换为 2D 图像的完整流程 |
| 顶点着色器 | Vertex Shader | 对每个顶点执行一次的 GPU 程序，计算顶点在屏幕上的位置 |
| 片元着色器 | Fragment Shader | 对每个像素执行一次的 GPU 程序，计算该像素的颜色 |
| 光栅化 | Rasterization | 将三角形转换为屏幕像素的过程 |
| 图元装配 | Primitive Assembly | 将顶点按指定方式连接成基本图形（点、线、三角形） |
| NDC | Normalized Device Coordinates | 归一化设备坐标，范围 [-1, 1]，超出会被裁剪 |
| 帧缓冲 | Framebuffer | GPU 渲染的目标区域，可以是屏幕或离屏纹理 |

## 坐标与变换

| 术语 | 英文 | 定义 |
|------|------|------|
| 世界坐标 | World Space | 所有物体在 3D 场景中的绝对位置 |
| 局部坐标 | Local Space | 物体相对于自身的坐标，原点在物体中心 |
| 变换矩阵 | Transform Matrix | 4×4 矩阵，用于平移/旋转/缩放物体 |
| MVP | Model-View-Projection | 模型变换→视图变换→投影变换的完整流程 |
| 四元数 | Quaternion | 用于表示旋转的数学工具，避免万向节锁 |
| 欧拉角 | Euler | 用 x, y, z 三个角度表示旋转，可能出现万向节锁 |
| 万向节锁 | Gimbal Lock | 欧拉角旋转时两个轴重合导致丢失一个自由度的问题 |

## Shader / GLSL

| 术语 | 英文 | 定义 |
|------|------|------|
| GLSL | OpenGL Shading Language | 运行在 GPU 上的着色器编程语言，类 C 语法 |
| ShaderMaterial | — | Three.js 中使用自定义着色器的材质类型 |
| attribute | — | 顶点着色器中每个顶点不同的数据（已被 WebGL2 废弃，但 Three.js 内部仍使用） |
| uniform | — | 所有顶点/片元相同的全局数据（时间、分辨率、纹理等） |
| varying | — | 顶点着色器传递到片元着色器的数据，GPU 自动插值 |
| UV 坐标 | UV Coordinates | 纹理坐标系统，u 从左到右 (0→1)，v 从下到上 (0→1) |
| 噪声 | Noise | 伪随机函数，用于生成自然的随机图案（云朵、火焰、地形等） |
| fBm | Fractal Brownian Motion | 多层噪声叠加，产生更丰富的细节 |

## 材质与光照

| 术语 | 英文 | 定义 |
|------|------|------|
| PBR | Physically Based Rendering | 基于物理的渲染，模拟真实世界的光照行为 |
| 粗糙度 | Roughness | 表面微观不平整程度，0=镜面，1=完全磨砂 |
| 金属度 | Metalness | 表面金属质感，0=非金属（塑料/木头），1=纯金属 |
| 环境光遮蔽 | AO (Ambient Occlusion) | 模拟角落和缝隙中光线难以到达的暗区 |
| 漫反射 | Diffuse | 光线射到粗糙表面后向各个方向均匀散射 |
| 镜面反射 | Specular | 光线在光滑表面上的定向反射，形成高光 |
| 菲涅尔效果 | Fresnel Effect | 视线与表面夹角越小时反射越强，边缘发光的基础 |
| 环境贴图 | Environment Map | 用于模拟周围环境反射的纹理 |
| HDR | High Dynamic Range | 高动态范围，存储比普通图片更宽的亮度范围 |

## 模型与动画

| 术语 | 英文 | 定义 |
|------|------|------|
| GLTF/GLB | GL Transmission Format | 3D 模型的标准格式，GLB 是二进制版本 |
| DRACO | — | Google 开发的 3D 网格压缩算法 |
| 骨骼动画 | Skeletal Animation | 通过骨骼驱动网格顶点变形的动画方式 |
| 蒙皮 | Skinning | 将网格顶点绑定到骨骼的过程 |
| 关键帧 | Keyframe | 动画中特定时间点的状态，中间帧由 GPU 插值计算 |
| 混合形状 | Blend Shape / Morph Target | 通过混合多个顶点位置来实现变形动画 |

## 性能

| 术语 | 英文 | 定义 |
|------|------|------|
| 实例化渲染 | Instanced Rendering | 用一次绘制调用渲染大量相同几何体的技术 |
| LOD | Level of Detail | 根据物体距离自动切换不同精度模型 |
| 视锥剔除 | Frustum Culling | 不渲染相机视野外的物体 |
| 绘制调用 | Draw Call | CPU 向 GPU 发送的一次渲染指令，越少越好 |
| 帧率 | FPS (Frames Per Second) | 每秒渲染的帧数，60fps 为流畅标准 |
| 内存泄漏 | Memory Leak | 不再使用的资源未释放，导致内存持续增长 |
