Demo 01：Hello Cube — 你的第一个3D场景

不需要理解 WebGL，不需要写 Shader，只需要 10 行核心代码，就能在浏览器中看到一个旋转的彩色立方体。 这就是 Three.js 的魔力——它帮你封装了所有底层细节。

🎯 本课目标：跑通第一个 Three.js 场景，感受"写几行代码就能出3D效果"的成就感。不理解每行代码完全没关系，后面会逐一拆解。

🎮 实时效果

index.html — 核心代码仅 10 行
// 1. 创建场景（舞台）
const scene = new THREE.Scene();

// 2. 创建相机（观众的眼睛）
const camera = new THREE.PerspectiveCamera(75, w/h, 0.1, 1000);

// 3. 创建渲染器（画笔）
const renderer = new THREE.WebGLRenderer();

// 4. 创建物体（立方体）
const cube = new THREE.Mesh(
  new THREE.BoxGeometry(1, 1, 1),  // 形状
  new THREE.MeshStandardMaterial({ color: 0x3b82f6 })  // 材质
);
scene.add(cube);

// 5. 渲染循环（每帧旋转 + 重绘）
function animate() {
  requestAnimationFrame(animate);
  cube.rotation.x += 0.01;  // 绕X轴旋转
  cube.rotation.y += 0.01;  // 绕Y轴旋转
  renderer.render(scene, camera);
}

📖 逐行拆解：Three.js 三大核心

每个 Three.js 场景都由 三个核心对象 组成，缺一不可：

对象类比作用忘记会怎样
Scene 场景 🎭 舞台 容纳所有3D物体的容器 物体无处可放，画布全黑
Camera 相机 👁️ 观众的眼睛 决定从哪个角度观察场景 看不到任何东西（视线不对）
Renderer 渲染器 🖌️ 画笔 把3D场景"画"到屏幕上 Scene和Camera全白费，屏幕无内容
💡 记忆口诀："舞台(Scene)上放了东西，观众(Camera)在某个位置看，画师(Renderer)把画面画出来。"

📖 Mesh（网格）：形状 + 材质 = 物体

Three.js 中所有可见的3D物体都是 Mesh。Mesh = Geometry（几何体/形状）+ Material（材质/外观）：

new THREE.Mesh(
  new THREE.BoxGeometry(1, 1, 1),   // ← Geometry：长什么样
  new THREE.MeshStandardMaterial({     // ← Material：什么颜色/质感
    color: 0x3b82f6
  })
)
概念类比常见类型
Geometry 几何体🏺 花瓶的"形状"BoxGeometry 立方体、SphereGeometry 球体、CylinderGeometry 圆柱体…
Material 材质🎨 花瓶的"釉色"MeshStandardMaterial 标准材质、MeshBasicMaterial 不受光材质…

⚠️ 常见踩坑

🚫 画布黑屏，什么都看不到？ 三步排查：
  1. 物体有没有 scene.add(cube)？忘了加 = 舞台空无一物
  2. 相机位置是否正确？camera.position.z = 5 确保相机离物体够近
  3. 有没有调用 renderer.render(scene, camera)？忘了渲染 = 画师没动笔
⚠️ 物体不旋转？ 检查 animate() 函数是否被调用，以及旋转代码是否在 renderer.render() 之前。

🏋️ 课后练习

  1. 改颜色：把立方体改成你喜欢的颜色（0xff0000 是红色，0x00ff00 是绿色）
  2. 改大小：把立方体改成 2x2x2 和 0.5x0.5x0.5，观察变化
  3. 加第二个物体：在场景中再添加一个球体，放在立方体旁边
  4. 只绕一个轴旋转：注释掉 X 轴旋转，只保留 Y 轴，观察效果
