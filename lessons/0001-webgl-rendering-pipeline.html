本课是整个 Three.js 学习之旅的基石。在调用 new THREE.Mesh() 之前，你需要理解 GPU 到底在做什么。 我们会手写原生 WebGL，画出人生第一个 GPU 渲染的三角形——这是所有 3D 场景的起点。

💡 为什么从原生 WebGL 开始？
Three.js 是对 WebGL 的封装。理解底层后，你排错时不会只看到黑屏，而是知道"着色器编译失败"还是"缓冲区没绑定"。这也是面试中渲染管线的高频考点。

1. WebGL 渲染管线：一张图看懂

GPU 渲染一个三角形，数据从 JavaScript 出发，经过 5 个阶段到达屏幕：

→
2

逐顶点：计算位置
→
3

三点 → 三角形
→
4
光栅化
三角形 → 像素
→
5
片元着色器
逐像素：上色
阶段做什么类比
① 顶点数据JS 把顶点坐标、颜色等数据传入 GPU 缓冲区把面粉和水准备好
② 顶点着色器GLSL 程序，对每个顶点执行一次，计算它在屏幕上的位置捏出饺子的形状
③ 图元装配把顶点按指定方式连接成基本图形（点、线、三角形）把饺子摆好
④ 光栅化将三角形转换为屏幕上的像素片段（fragment）决定每个饺子占几个格子
⑤ 片元着色器GLSL 程序，对每个像素执行一次，决定该像素的颜色给每个饺子上色
⚠️ 关键认知：顶点着色器决定"形状在哪"，片元着色器决定"像素什么颜色"。这两个着色器是你必须自己写的（GPU 不会帮你做），其余步骤 GPU 自动完成。

2. 第一个 WebGL 程序：绘制三角形

下方是完整的 WebGL 代码，绘制一个彩色三角形。左边是源码，右边是实时渲染结果。

main.js🔄 重置
// 1. 获取 Canvas & WebGL 上下文
const canvas = document.getElementById('triCanvas');
const gl = canvas.getContext('webgl');

// 2. 顶点着色器 GLSL
const vsSource = `
  attribute vec4 a_position;
  void main() {
    gl_Position = a_position;
  }`;

// 3. 片元着色器 GLSL
const fsSource = `
  precision mediump float;
  uniform vec4 u_color;
  void main() {
    gl_FragColor = u_color;
  }`;

// 4. 编译着色器
function compileShader(gl, source, type) {
  const shader = gl.createShader(type);
  gl.shaderSource(shader, source);
  gl.compileShader(shader);
  if (!gl.getShaderParameter(shader, gl.COMPILE_STATUS)) {
    console.error(gl.getShaderInfoLog(shader));
  }
  return shader;
}

// 5. 链接程序
const program = gl.createProgram();
gl.attachShader(program, compileShader(gl, vsSource, gl.VERTEX_SHADER));
gl.attachShader(program, compileShader(gl, fsSource, gl.FRAGMENT_SHADER));
gl.linkProgram(program);
gl.useProgram(program);

// 6. 创建缓冲区 & 写入顶点数据
const vertices = new Float32Array([
   0.0,  0.5,  // 顶点 1 (上)
  -0.5, -0.5,  // 顶点 2 (左下)
   0.5, -0.5   // 顶点 3 (右下)
]);
const buffer = gl.createBuffer();
gl.bindBuffer(gl.ARRAY_BUFFER, buffer);
gl.bufferData(gl.ARRAY_BUFFER, vertices, gl.STATIC_DRAW);

// 7. 启用顶点属性
const aPos = gl.getAttribLocation(program, 'a_position');
gl.enableVertexAttribArray(aPos);
gl.vertexAttribPointer(aPos, 2, gl.FLOAT, false, 0, 0);

// 8. 设置颜色 & 绘制
const uColor = gl.getUniformLocation(program, 'u_color');
gl.uniform4f(uColor, 0.2, 0.6, 1.0, 1.0); // 蓝色
gl.clearColor(0.06, 0.06, 0.12, 1.0);
gl.clear(gl.COLOR_BUFFER_BIT);
gl.drawArrays(gl.TRIANGLES, 0, 3);
实时渲染
🔍 逐行阅读指南：代码中的 8 个步骤，每一个都对应渲染管线的一个阶段。试着改一下 gl.uniform4f 的第 2-4 个参数（R, G, B），看看三角形颜色变化。

4. GLSL 着色器语言速览

GLSL（OpenGL Shading Language）是运行在 GPU 上的 C 语言变体。以下是它与 JavaScript 的关键区别：

特性JavaScriptGLSL
变量声明let x = 1;float x = 1.0; — 必须声明类型
数字1 自动整数/浮点1 是 int，1.0 是 float，必须严格区分
向量类型无vec2(x,y) vec3(x,y,z) vec4(x,y,z,w)
矩阵类型无mat2 mat3 mat4
精度声明不需要片元着色器必须声明：precision mediump float;
保留字let const varattribute varying uniform（限定符，不是声明符）
🚫 高频踩坑：在 GLSL 中写 let x = 1; 或 1 + 0.5（整数和浮点数混用）会导致编译失败。GLSL 不能用 let、console.log、模板字符串等 JS 特性。

5. NDC 坐标系统

WebGL 使用的是 归一化设备坐标（NDC，Normalized Device Coordinates）：

X 轴：左 -1 → 右 +1  |  Y 轴：下 -1 → 上 +1  |  超出范围 = 被裁剪消失
⚠️ 常见错误：顶点坐标超出 [-1, 1] 范围后，图形会被裁剪，表现为"画布上什么都没有"。如果三角形不显示，先检查顶点坐标是否在 NDC 范围内。
